## 1. 프로젝트 개요

- **프로젝트명**: CrewLife (가칭)
- **목표**: 복잡한 승무원 스케줄표를 AI로 디지털화하고 동료·지인과 실시간 공유
- **핵심 타겟**: 국내외 항공사 객실승무원 및 지인
- **운영 전략**: '개인 프로젝트' 톤앤매너로 겸업 리스크 최소화, 유저 접근성↑

## 2. 주요 기능

### 2.1 AI 스케줄 인식 및 수동 정정

- **AI OCR 파싱**: Claude Sonnet 4.6 (Vision) API로 캡처 이미지에서 스케줄 추출
- **비용 최적화**: AI는 **최초 입력 1회만** 호출
- **공항 DB 매칭**: 추출된 IATA/ICAO 코드 → 사전 구축 공항 DB와 매칭 → 공항명, 도시명, 시차 보강
- **항공사 자동 감지**: 편명(예: `KE017`, `OZ201`)의 앞 2자리 IATA 코드로 소속 항공사 자동 식별 → 프로필의 "소속 항공사"는 회원가입 시 묻지 않고 첫 스케줄 등록 시 자동 채움
- **수동 정정·검수**: AI 결과를 유저가 직접 수정 가능한 UI → 최종 확인 후 DB 저장
- **빠른 수동 입력**: 이미지 업로드 없이도 바텀시트로 즉석 입력 (삼성 캘린더 느낌) — Phase 2에서 연결

### 2.2 스케줄 꾸미기 및 관리

- **기본 UI**: 달력(Calendar) 형태 메인 화면
- **커스텀**: 스티커, 메시지 작성, 자유 그리기
- **항공사 브랜딩**: 감지된 항공사별 고유 컬러/문양을 스케줄 카드에 표시. 이미지 리소스는 `app/assets/airlines/` 폴더에 수동 투입(파일명 = IATA 소문자 코드 + `.png`, 예: `ke.png`, `oz.png`).
- **외부 캘린더 동기화**:
  - Google Calendar — OAuth 로그인 후 캘린더 선택 UI (구글 캘린더 프로필 선택 느낌)
  - Apple Calendar (EventKit) — iOS 네이티브 권한 → 캘린더 선택
  - 동기화 방향: CrewLife → 외부 (1-way, Phase 2 후반)

### 2.3 공유 및 커뮤니티

- **연락처 기반 친구 공유**: 전화번호로 친구 탐색 후 비행 일정 공유
- **외부 공유**: 이미지 카드(인스타 스토리용) / 텍스트 변환 → 카카오톡 등
- **친구 초대**: 초대 코드 시스템 → 신규 유입
- **활동 소식**: 친구 비행 전 '응원 보내기' (메시지·이모티콘)

### 2.4 보안 및 인증

- **회원가입/로그인**: **카카오톡 로그인 단일화** (Phase 1 후반)
  - Kakao Developers 앱 등록 → REST/Native 앱키 발급
  - Supabase Auth custom provider 또는 Edge Function으로 카카오 토큰 → Supabase 세션 교환
  - 이메일 가입은 개발용으로만 유지, 프로덕션에선 비활성
- **앱 진입 보안**: Face ID / 지문 / 4~6자리 비밀번호 (Phase 4)
- **승무원 인증**:
  - **A**: 스케줄표 패턴 매칭(AI) → 자동 승무원 인증
  - **C**: 기존 인증 유저의 초대 코드 → 신뢰 기반 인증
  - **B (추후)**: 사원증 사진 인증 (민감정보 자동 마스킹)

## 3. 기술 스택

| 구분 | 기술 | 비고 |
|---|---|---|
| 프레임워크 | Flutter | iOS/Android 크로스 플랫폼 |
| 백엔드/DB | Supabase | Auth, DB, Edge Functions |
| AI 엔진 | Claude Sonnet 4.6 | 이미지 분석·데이터 추출 |
| 인증 | Kakao Login | Supabase Auth와 연동 |
| 캘린더 연동 | Google Calendar API / Apple EventKit | 1-way 내보내기 |
| 광고/수익 | Google AdMob | 배너·보상형 |

### 주요 테이블 (Supabase)

- **airports**: IATA, ICAO, 공항명(국문), 도시명, 타임존
- **airlines** (신규): IATA 2자리 코드, ICAO 3자리 코드, 항공사명(ko/en), 브랜드 컬러, 문양 파일명
- **profiles**: UID, 카카오 ID, 전화번호, 인증 상태, 소속 항공사(자동 감지), 프리미엄 여부
- **schedules**: 유저별 비행 일정, 편명, 감지된 항공사 코드, 수정 데이터, 꾸미기 정보

## 4. 앱 메뉴 구성 (3슬롯 커스텀 하단바 + 사이드 드로어)

기존 4단 NavigationBar → **중앙 빠른 입력 + 좌/우 토글/드로어** 구조로 재설계.

```
┌─────────────────────────────┐
│         메인 영역             │
│  (내 스케줄 달력 / 친구 화면)    │
├───────┬───────────┬─────────┤
│ 친구   │  + 입력    │  설정   │
│ (네이비)│(중앙 FAB) │ (네이비)│
└───────┴───────────┴─────────┘
```

### 4.1 좌측 슬롯 — 친구 / 내 스케줄 토글

- **기본(내 스케줄 화면일 때)**: "친구" 버튼, 네이비 배경, 사람 아이콘
- **누르면**: 왼쪽에서 **네모난 사이드 패널**이 슬라이드 인 → 친구 목록 드로어 (Drawer)
- 드로어 내부에서 친구 선택 시 본 화면이 "친구 스케줄" 모드로 전환
- 전환 후에는 좌측 슬롯이 **"내 스케줄"** 버튼으로 바뀜 (아이콘도 달력 아이콘으로 교체) → 다시 누르면 내 스케줄 화면으로 복귀

### 4.2 중앙 슬롯 — 빠른 스케줄 입력

- 삼성 캘린더의 '일정 추가' 느낌 — 중앙 원형/캡슐 FAB
- 탭 → 하단 시트 (bottom sheet)로 빠른 입력 폼: 날짜, 편명, 출발/도착, 메모
- 편명 입력 즉시 항공사 자동 감지 → 헤더에 브랜드 컬러/문양 프리뷰
- AI OCR과 별개 경로 (수동 입력은 Claude 호출 없이 DB 직저장)

### 4.3 우측 슬롯 — 설정 사이드 드로어

- "설정" 버튼, 네이비 배경, 기어 아이콘
- 누르면 **오른쪽에서 네모난 사이드 패널**이 슬라이드 인 → 설정 드로어 (Drawer)
- 드로어 내부: 프로필, 카카오 재로그인, 생체 잠금, 캘린더 연동, 초대 코드, 구독, 앱 정보
- 메인 화면 전환 없이 오버레이로 동작

### 4.4 활동 소식

- 별도 탭 제거 → 상단 AppBar의 알림 벨 아이콘으로 이동 (Phase 3에서 피드 화면 연결)
